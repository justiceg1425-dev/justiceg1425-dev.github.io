LLM Test Case Generator

A Streamlit application that parses BRDs and API specifications, generates structured test cases, stores version history, and exports test assets in formats useful for QA teams.

Problem

Manual test case authoring is slow and inconsistent.

When teams receive BRDs, PDFs, text requirements, or API specs, testers still need to extract flows, preconditions, edge cases, and expected results. That work is valuable, but the first-pass drafting can consume time that could be spent reviewing quality and coverage.

What the tool solves

  • Accepts requirement documents and YAML API specs.
  • Parses structured requirements into flows, actors, preconditions, and triggers.
  • Generates reviewable test cases through LLM prompts and deterministic helpers.
  • Maintains version history per uploaded file.
  • Exports in formats that can move into downstream QA tools.

Feature depth

From requirements to structured QA assets.

Document intake

Supports upload of BRDs and requirement files such as DOCX, PDF, TXT, plus YAML API specifications for endpoint-oriented test generation.

Requirement parsing

Extracts high-level requirement structure such as title, description, actors, flows, alternative paths, preconditions, postconditions, and API endpoint details.

LLM generation

Uses prompt-driven generation to produce detailed test cases with description, steps, expected results, type, and API-specific context.

Review and filtering

Generated cases can be viewed, expanded/collapsed, searched by keyword, filtered, restored from history, and compared against previous versions.

Exports

Exports generated test cases to JSON, Excel, QMetry CSV, and Gherkin JSON so output can move into test management or automation workflows.

QMetry path

Includes optional integration provisions for pushing generated test cases into QMetry, which makes it closer to a QA operations tool than a standalone generator.

Architecture

Streamlit workflow with parser, generator, exporter, and history modules.

The tool is structured around a clear pipeline: ingest a file, parse requirements, generate cases, store versions, then export or integrate with downstream QA platforms.

Setup and use

  1. Create a Python environment and install project requirements.
  2. Run the Streamlit application locally.
  3. Log in using configured user access.
  4. Upload a BRD, PDF, TXT, or YAML API specification.
  5. Review parsed requirements, generate test cases, and inspect the output.
  6. Export to JSON, Excel, QMetry CSV, or Gherkin JSON.
